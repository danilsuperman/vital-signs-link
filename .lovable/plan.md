## План: Share Center (поделиться медицинским контекстом)

Доработка в три шага. Сфокусирована на функции «Поделиться» по ТЗ — остальные крупные доработки (digital twin, обращения, команда, ещё) идут следующими итерациями.

### 1. Ядро: компонент Share Center
Создать `src/components/share-dialog.tsx` — универсальный диалог, открывается из любой точки приложения с параметрами:
- `scope`: `"full" | "case" | "labs" | "documents" | "temp"` — предвыбранный уровень
- `caseTitle?`: название кейса (если шарим конкретный случай)
- `context?`: краткое описание (например, «Варикоз #245» или «Анализы крови»)

Содержимое диалога (по ТЗ п.6–7):
1. **Заголовок**: «Поделиться медицинским контекстом» + подзаголовок «Врач увидит клиническую картину, а не сырые данные»
2. **5 уровней доступа** (radio-карточки с иконкой, названием, описанием):
   - Полный медицинский доступ
   - Активный случай
   - Только анализы
   - Только документы
   - Ограниченный просмотр (48 ч)
3. **Канал отправки**: табы Ссылка / Email / WhatsApp
4. **Срок действия**: select (24ч / 48ч / 7 дней / без срока) — авто-устанавливается для temp
5. **Кнопка «Создать ссылку»** → генерирует `secure.lify.health/case/XXXX-XXXX` (рандом 8 символов), показывает превью ссылки + кнопки «Скопировать», «Открыть QR», «Отправить»
6. **Превью что увидит врач** — краткий list: Контекст пациента → Активные cases → Таймлайн → ИИ-выводы → Рекомендации → Файлы (фильтруется по уровню)

Под капотом: localStorage-стор `src/lib/share-links-store.ts` (id, scope, caseId?, channel, expiresAt, createdAt). На «Создать» — toast + ссылка отображается.

### 2. Кнопки «Поделиться» в интерфейсе
- **Главная (`src/routes/index.tsx`)**: в шапке (рядом с прогрессом медкарты) — крупная кнопка `[Поделиться медкартой]` → открывает ShareDialog с scope=`full`.
- **Карточки кейсов** (UserCaseCard и CaseCard на главной): кнопка-иконка `Share2` в правом верхнем углу → scope=`case`, передаём название кейса.
- **Лечение (`src/routes/treatment.tsx`)**: в заголовке каждого активного кейса — `[Поделиться кейсом]` → scope=`case`.
- **Здоровье (`src/routes/health.tsx`)**: возле блока анализов — `[Поделиться анализами]` → scope=`labs`.
- **Ещё → Файлы (когда появится)**: `[Поделиться документами]` → scope=`documents`.

В этой итерации добавлю кнопки в местах, где сейчас уже есть соответствующие блоки (главная, treatment, health). Кнопки для «Ещё»/Файлы оставлю на следующую итерацию вместе с переработкой раздела «Ещё».

### 3. Технические детали
- Шрифт радиус кнопок: уже `rounded-xl` по проекту — сохраняем.
- Использовать существующие shadcn: `Dialog`, `RadioGroup`, `Tabs`, `Select`, `Button`, `Badge`, `sonner` для копирования.
- Без бэкенда: всё в localStorage. Реальные ссылки/QR — заглушки, но визуально полные.
- Никаких изменений в auth/server-функциях.

### Изменяемые файлы
- create `src/components/share-dialog.tsx`
- create `src/lib/share-links-store.ts`
- edit `src/routes/index.tsx` (кнопка «Поделиться медкартой», иконка share в кейсах)
- edit `src/routes/treatment.tsx` (кнопка в каждом кейсе)
- edit `src/routes/health.tsx` (кнопка возле анализов)
